# Nayara Resorts: Claude Code handoff

This folder is a self-contained brief for continuing the Nayara Gardens organic strategy work in Claude Code.

## How to use it

1. Unzip this folder somewhere on your machine (or drop it into your existing repo, for example a clone of `albertghitis-netizen/nayara-bespoke-experiences`).
2. Open Claude Code in this folder (run it from this directory, or point the desktop app here).
3. Claude Code automatically reads `CLAUDE.md` at the start of every session, so it will be briefed on the whole project without you re-explaining: the standing rules, the design system, the domain architecture, the four-page landing plan, the data findings, and what is already underway on the name reconnection.
4. Start working. For example: "Build the Gardens family page per the plan," or "Write the Hotel and FAQ schema for the Gardens homepage," or "Draft the Which Nayara comparison page."

## What is here

- `CLAUDE.md` — the master context file. Read first. Claude Code loads this automatically.
- `deliverables/` — finished work products: the executive briefing, the integrated demand strategy, the search demand plan, and the Tented paid-search audit.
- `reference/` — background documents: the external SEO audit, the strategy critique, and the integrated growth strategy deck text.

## Notes

- A Claude.ai Project and Claude Code are separate surfaces, so the conversation history does not transfer automatically. `CLAUDE.md` is the bridge: it carries the context forward.
- Keep `CLAUDE.md` updated as decisions change. It is the single source of truth for this work.
- Standing rules to remember: no em dashes, editorial is the "Journal," real photography only, "Check availability" not "Book now," and no changes to live accounts or listings without explicit go-ahead.
